---

1. 다음 명령들을 사용하여 현재 변경된 파일 목록을 확인합니다:
   - `git diff --name-only` (수정된 파일)
   - `git diff --staged --name-only` (스테이징된 파일)
   - `git status --porcelain` (추적되지 않은 신규 파일 포함)
2. 변경된 파일 중 문서 파일(*.md, *.txt 등)만 필터링합니다.
3. 각 문서 파일의 내용을 읽고 다음 항목을 검토합니다:
   - 오탈자 (맞춤법 오류)
   - 부정확한 표현 (어색한 문장, 중복 표현 등)
   - 일관성 없는 용어 사용
4. 발견된 문제점을 자동으로 수정합니다.
5. 수정된 내용을 사용자에게 요약하여 보여줍니다.

# 문서 파일 필터링

다음 확장자를 가진 파일을 문서로 간주합니다:
- `.md` (Markdown)
- `.txt` (텍스트)
- `.rst` (reStructuredText)

# 검토 기준

## 한국어 문서
- 맞춤법 및 띄어쓰기
- 조사 사용의 정확성
- 문장 종결어미 일관성

## 영어 문서
- Spelling errors
- Grammar issues
- Punctuation consistency

# 수정 원칙

- 원문의 의도를 최대한 보존합니다
- 스타일보다는 명확한 오류에 집중합니다
- 수정 전후를 명확히 표시하여 사용자가 확인할 수 있도록 합니다
